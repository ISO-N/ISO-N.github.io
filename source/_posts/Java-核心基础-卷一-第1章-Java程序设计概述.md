## 1.1 Java 程序设计平台

- Java 不仅是一门编程语言，更是一个完整的**平台**，包含庞大的库、大量可复用代码以及一个提供安全性、跨操作系统可移植性和自动垃圾回收等服务的执行环境
- Java 的优势在于其丰富的**标准类库（API）**，涵盖 GUI、网络、数据库、并发等众多领域

## 1.2 Java "白皮书"关键术语

Java 设计者发布了一组**11 个关键术语**来总结 Java 的设计目标和特性：

### 简单性（Simple）

- 语法是 C++ 的"纯净版"，去掉了头文件、指针运算、结构体、联合体、操作符重载、虚基类等复杂特性
- 开发环境体积小，易于在小型设备上运行

### 面向对象（Object-Oriented）

- Java 的面向对象特性与 C++ 相当，主要区别在于 Java 的**多重继承**被更简洁的**接口（interface）**概念取代
- Java 比 C++ 更注重对象间的交互和接口设计

### 分布式（Distributed）

- Java 具备丰富的网络处理类库，可通过 URL 打开和访问网络对象，如同访问本地文件一样方便
- 支持远程方法调用（RMI），使分布式应用开发更加简单

### 健壮性（Robust）

- Java 采用**早期问题检测**、后期动态检查以及消除容易出错的机制来保证程序可靠运行
- 与 C/C++ 不同，Java 的指针模型**杜绝了内存覆盖和数据损坏**的可能

### 安全性（Secure）

- Java 从一开始就被设计为能防止各种攻击：
    - 运行时堆栈溢出（蠕虫常用手段）
    - 在自身处理空间之外破坏内存
    - 未经授权读写文件

### 体系结构中立（Architecture-Neutral）

- Java 编译器生成与特定计算机体系结构无关的**字节码（bytecode）**
- 字节码可在任何实现了 Java 虚拟机（JVM）的机器上运行
- 核心理念：**"Write Once, Run Anywhere"**

### 可移植性（Portable）

- 基本数据类型大小固定（如 `int` 永远为 32 位），消除了不同平台间数据类型差异
- 二进制数据以固定格式存储和传输，字符串使用标准的 **Unicode** 编码

### 解释型（Interpreted）

- Java 字节码可以在任何有 JVM 的机器上被**解释执行**
- 链接过程轻量且增量化，加速了开发周期

### 高性能（High-Performance）

- 虽然解释型字节码速度一般够用，但对于性能敏感的场景，字节码可以在运行时由**即时编译器（JIT）**翻译为本地机器码
- 现代 JVM 的 JIT 编译技术使 Java 在许多场景下性能接近甚至媲美 C++

### 多线程（Multithreaded）

- Java 是最早在语言层面内置**多线程支持**的主流编程语言之一
- 多线程编程简单、稳定，能充分利用多核处理器优势

### 动态性（Dynamic）

- Java 能够**动态加载类和库**，适应不断变化的环境
- 可以在运行时添加新方法和实例变量，方便进行类的查找和动态调用

## 1.3 Java applet 与 Internet

- **Applet** 是最初让 Java 广为人知的技术，是一种可在网页浏览器中运行的小程序
- 用户从 Internet 下载字节码并在浏览器中运行（通过沙箱安全模型提供保护）
- 由于安全问题和其他技术（如 HTML5）的兴起，Applet 已被逐步淘汰

## 1.4 Java 发展简史

| **版本** | **年份** | **重要特性** |
| --- | --- | --- |
| 1.0 | 1996 | 语言本身发布 |
| 1.1 | 1997 | 内部类 |
| 1.2 | 1998 | Swing GUI、集合框架 |
| 1.3 | 2000 | 性能优化（HotSpot JVM） |
| 1.4 | 2002 | XML 处理、正则表达式 |
| 5.0 | 2004 | 泛型、注解、自动装箱、枚举、for-each 循环 |
| 6 | 2006 | 性能提升、库改进 |
| 7 | 2011 | try-with-resources、钻石操作符、switch 支持字符串 |
| 8 | 2014 | **Lambda 表达式**、Stream API、新的日期/时间 API |
| 9 | 2017 | 模块化系统（Jigsaw） |
| 10+ | 2018+ | 每 6 个月发布新版本，var 局部变量推断等 |

## 1.5 关于 Java 的常见误解

<aside>
⚠️

**需要注意的几点常见误解：**

- ❌ Java 是 HTML 的扩展 → ✅ Java 是独立的编程语言，与 HTML 无关
- ❌ Java 很容易学 → ✅ Java 语法相对友好，但掌握整个生态和库需要大量学习
- ❌ Java 很慢 → ✅ 现代 JIT 编译器使 Java 性能非常出色
- ❌ 所有 Java 程序都在网页中运行 → ✅ Java 主要用于**服务器端**、移动端（Android）和桌面应用
- ❌ Java 不安全 → ✅ Java 安全模型经过多次加强，已非常完善
</aside>
