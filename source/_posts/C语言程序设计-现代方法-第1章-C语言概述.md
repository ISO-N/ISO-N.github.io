## 1.1 C语言的历史

- C语言由 **Dennis Ritchie** 于 1970 年代初在贝尔实验室开发，最初用于实现 **UNIX** 操作系统
- C语言的前身：
    - **BCPL**（Martin Richards，1967）
    - **B语言**（Ken Thompson，1970）——BCPL 的精简版，用于早期 UNIX
    - **C语言**（Dennis Ritchie，1971-1973）——在 B 语言基础上增加了**数据类型**等特性

## 1.2 C语言的优缺点

### 优点

- **高效性**：贴近底层硬件，生成的代码运行速度快、占用资源少
- **可移植性**：C 编译器广泛存在于各种平台，程序容易移植
- **功能强大**：提供丰富的数据类型和运算符，能直接操作位、指针、内存
- **灵活性**：对程序员的限制少，自由度高
- **标准库**：提供了大量实用的标准库函数
- **与 UNIX 集成**：C 语言与 UNIX/Linux 系统紧密结合

### 缺点

- **容易隐藏错误**：C 语言的灵活性意味着某些错误不会被编译器捕获（如指针越界、类型不匹配等）
- **可读性差**：灵活的语法容易写出难以理解的代码
- **难以修改**：大型 C 程序如果设计不当，后期维护困难

## 1.3 标准化

- **K&R C**（1978）：Brian Kernighan 和 Dennis Ritchie 合著的 *The C Programming Language* 成为事实标准
- **C89/C90**（即 **ANSI C** / **ISO C**）：1989 年 ANSI 发布，1990 年 ISO 采纳（ISO/IEC 9899:1990），是第一个正式标准
- **C99**（1999）：ISO/IEC 9899:1999，新增了诸多特性：
    - `//` 单行注释
    - `long long int` 类型
    - `_Bool` 类型
    - 变长数组（VLA）
    - 声明可以不在块的开头
    - `<stdbool.h>`、`<stdint.h>` 等新头文件
- **C11**（2011）：进一步演进，增加了多线程支持、泛型选择等

## 1.4 基于C的语言

C语言深刻影响了许多后续语言：

- **C++**：在 C 基础上增加了面向对象、模板、异常等特性
- **Java**：语法大量借鉴 C/C++，但去掉了指针等底层操作
- **C#**：微软推出，语法受 C/C++/Java 影响
- **Perl**：大量继承了 C 的语法风格

> 学好 C 语言，可以为学习上述语言打下坚实基础。
